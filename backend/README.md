# DysLearnia Backend

The FastAPI-powered core engine for DysLearnia. It handles the dynamic pipeline execution, block registry, vector grounding (RAG), and knowledge graph interactions that power the visual content transformation platform.

## 🚀 Overview

DysLearnia is a visual learning pipeline builder, and this backend is responsible for processing those pipelines. It takes a topological graph from the React Flow frontend, resolves dependencies, and dynamically runs "Blocks" — isolated units of work powered by local models and AI agents.

### Key Components

1. **Topological Pipeline Engine**: Resolves dependencies between connected blocks and runs them in the correct order, streaming WebSocket events back to the client.
2. **Block Registry System**: A dynamic `@register_block` decorator-based system built on the `IBlock` interface. New features can be added by simply dropping a new Python file into the `blocks/` directory.
3. **Local AI Fleet**: Connects seamlessly with local and containerized AI:
   - **Ollama**: Generative LLM tasks using Qwen2.5-3B-Instruct.
   - **Whisper**: Local speech-to-text for audio formatting and voice pipeline building.
   - **Kokoro-82M**: Local text-to-speech engine.
4. **Data Layer**:
   - **Supabase**: Auth, Flow saves, Pipeline caching, and File Storage.
   - **Qdrant Cloud**: Vector similarity search for RAG-grounded responses.
   - **Neo4j**: Graph database for the dynamic learning connection engine.

## 📁 Repository Structure

```text
backend/
├── agents/             # Legacy standalone AI agents
├── api/                # FastAPI routing logic (endpoints & websockets)
├── blocks/             # Implementation of the execution blocks (IBlock)
│   ├── inputs/         # Source extractors (PDF, PPTX, Video)
│   ├── transform/      # Mutators (Summarizer, Flashcards, Quiz)
│   └── output/         # Format unifiers (PDF, MP3, MP4)
├── core/               # Engine logic, state management, and graph evaluation
├── utils/              # Wrappers around external systems (Ollama, PDF parsing)
├── main.py             # FastAPI application entrypoint
└── requirements.txt    # Project dependencies
```

## 🛠️ Setup & Execution

### Prerequisites

- Python 3.13+
- `uv` package manager (recommended for speed)

### Installation

1. Create a virtual environment and activate it:

   ```bash
   uv venv
   source .venv/bin/activate  # On Windows: .venv\Scripts\activate
   ```

2. Install the dependencies:

   ```bash
   uv pip install -r requirements.txt
   ```

3. Configure your Environment Variables by copying the `.env.example` file (ensure you have Supabase and Qdrant Cloud credentials).

### Running the API

Start the backend server in development mode:

```bash
fastapi dev main.py
# Or using generic uvicorn:
# uvicorn main:app --reload
```

The system will start running on `http://localhost:8000`. You can test endpoints via the autogenerated Swagger documentation at `http://localhost:8000/docs`.

## 🧩 The IBlock System

Every block in the workspace follows a strict typed input-output envelope known as `BlockData`. For instance, when creating a new pipeline feature, you just need to inherit from the `IBlock` interface:

```python
from backend.blocks.base import IBlock, BlockData, BlockDescription
from backend.blocks.registry import register_block

@register_block
class MyCustomBlock(IBlock):
    description = BlockDescription(
        name="custom_block",
        display_name="Custom Effect",
        # ...
    )

    async def execute(self, params: dict, context: dict, input_data: BlockData) -> BlockData:
        # Transformation logic goes here
        # E.g., make an Ollama call, index to Qdrant, etc.
        return new_data
```

The system automatically discovers the block and exposes it via `GET /api/blocks`, populating the frontend UI canvas instantly.
